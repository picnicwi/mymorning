Section 2

How the AI Brain Works

The core of Claude is a Large Language Model (LLM). Think of it as an incredibly well-read assistant that has processed billions of pages of text and learned patterns of language and knowledge.

🪟 The Context Window — Claude's Working Memory

Everything Claude can "see" in one conversation lives inside a context window — like a big notepad that holds the full conversation. Claude reads it all at once each time it responds.

System You are Claude, a helpful AI in Cowork mode. You have access to files, bash, web search… [hundreds of lines of instructions]
User "Create a presentation about our Q1 results"
Think The user wants a PowerPoint. I should read the pptx skill first, then ask clarifying questions about the content…
Action → Read file: /skills/pptx/SKILL.md
Result [file contents returned: "Use python-pptx library…"]
Answer Sure! Before I start, could you tell me: Who is the audience, and what key metrics should I highlight?
💡 Analogy

Imagine a chef reading a recipe card. The recipe card is the context window — it contains the restaurant's rules, the customer's order, any notes from previous courses, and the chef's own running thoughts. The chef reads the whole card before every action. The bigger the card, the more the chef can remember at once.

📚

What the Model "Knows"

Claude was trained on a huge dataset up to a knowledge cutoff date. This training baked in language, facts, reasoning patterns, and skills. Think of it as education — Claude studied everything and now holds it as intuition, not a searchable database.

Language patterns World knowledge Coding skills Reasoning
🔢

Tokens, Not Words

Claude doesn't read letters or words directly — it reads tokens, small chunks of text (roughly ¾ of a word each). Every token gets processed through layers of math called attention, which figures out how each token relates to all others. This is how meaning emerges.

"Hello" = 1 token "understanding" ≈ 2 tokens ~200k tokens per context
Section 3

The Thinking Process (The Agent Loop)

For complex tasks, Claude doesn't respond instantly. It goes through a multi-step loop — thinking, acting, observing, and thinking again — until the task is done.

1

Read the Context

Claude reads your message + all instructions + conversation history.

2

Reason & Plan

Claude thinks through what's needed. This reasoning happens before writing any response.

3

Take an Action (Tool Call)

If needed, Claude calls a tool — read a file, run code, search the web.

4

Observe the Result

The tool's output is added back to the context window. Claude now "knows" the result.

5

Respond or Loop Again

If the task is done, Claude writes a final response. If not, it goes back to Step 2.

🔁 Steps 2–4 repeat as many times as needed
🤔

Why Does Claude "Think" First?

Modern AI models produce better answers when they reason step-by-step before answering — similar to how writing a draft helps you think. This is sometimes called Chain-of-Thought reasoning. Claude works through the problem internally before committing to an answer.

🔄

Why Multiple Rounds?

Real tasks are rarely one-shot. Creating a PowerPoint means: read skill instructions → write Python code → run it → check the output → fix errors → save the file. Each step informs the next.

🚦

When Does It Stop?

Claude decides when the task is complete. It checks whether its actions achieved the goal, then delivers a final answer to you — often with a link to a created file.

Section 4

How Claude Takes Actions

Claude by itself can only generate text. To do things — create files, run code, browse the web — it uses tools. Each tool is a capability plug-in that extends what Claude can accomplish.

📄

Read Files

Opens and reads any file on your computer or workspace.

✏️

Write & Edit Files

Creates new files or modifies existing ones (Word, Excel, etc.).

💻

Run Code (Bash)

Executes Python, JavaScript, or shell commands in a sandbox.

🌐

Web Search

Searches the internet for up-to-date information.

📡

MCP Connectors

Connects to external apps: Notion, Slack, Asana, and more.

🖥️

Browser Control

Navigates websites, fills forms, and extracts data from the web.

📁

Workspace Access

Reads and writes to your selected folder on your computer.

📋

Task Tracker

Maintains a visible to-do list so you can follow Claude's progress.

💡 Analogy

Think of Claude as an office assistant with a toolbelt. The assistant's brain (the LLM) decides what to do. The tools are like equipment: a calculator, a phone, a printer, a filing cabinet. The assistant picks the right tool, uses it, checks the result, and moves on — all on your behalf.

Section 5

Claude Cowork — Full Architecture

Cowork adds several layers on top of the base AI model to make it a practical desktop co-worker. Here's how the layers stack together:

👤

You (The User)

Types a message in the Cowork desktop app. Can also connect a folder, upload files, or install plugins.

↓
📜

System Prompt + Skills

Before Claude even sees your message, a detailed system prompt injects Cowork's rules, your identity, available tools, and instructions. Skills are pre-loaded best-practice guides Claude reads to do certain tasks (like making a .pptx or .docx) with high quality.

↓
🧠

Claude AI Model (The Brain)

The large language model at the core. Reads all context, reasons through the request, decides what actions to take, and generates responses. This is Anthropic's Claude (claude-sonnet or claude-opus).

↓
🔧

Tools Layer (Claude Agent SDK)

The infrastructure that lets Claude call tools. When Claude decides to "read a file" or "run Python", the Agent SDK routes that call to the right tool and returns the result back into the context window.

↓
🔌

MCP Servers & Plugins

Model Context Protocol (MCP) servers are bridges to external services — Notion, Slack, Google Drive, databases, etc. Plugins bundle MCPs + Skills + Tools together for specific workflows (like finance, data analysis, or productivity).

↓
💾

Your Computer / Outputs Folder

Final files created by Claude (reports, spreadsheets, slides) are saved to your selected outputs folder — accessible directly from your desktop.

🎯

What Makes Cowork Different from a Chatbot?

A regular chatbot only returns text. Cowork is an agent — it can plan multi-step tasks, use real tools, create real files on your computer, connect to real apps, and loop until the job is truly done. It's like upgrading from chatting to delegation.

🔐

What Does the Prompt Actually Do?

The system prompt is the invisible backbone. It tells Claude who it is, what tools are available, how to behave, what file paths to use, and how to present results. Think of it as an employee handbook read before every response.

Putting It All Together

Claude Cowork is a layered system: a powerful AI brain, guided by a detailed prompt, extended by tools and plugins, and looping through think → act → observe until your task is complete.

🧠

The LLM

Trained on billions of texts. Reads your full context and generates reasoned responses.

📜

The Prompt

System-level instructions that shape Claude's identity, capabilities, and behaviour.

🔧

The Tools

File access, code execution, web search, and app connectors that let Claude act in the world.

🔄

The Loop

Think → Act → Observe → Repeat. Complex tasks are solved step by step, not all at once.
